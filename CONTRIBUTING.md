# Contributing to MCP Security Research

Thank you for your interest in contributing to this security research project!

## Code of Conduct

This project is intended for educational and defensive security research purposes only. By contributing, you agree to:

- Use this research responsibly and ethically
- Not use the demonstrated vulnerabilities for malicious purposes
- Help improve security awareness and defensive measures

## How to Contribute

1. **Fork the repository**
2. **Create a feature branch** (`git checkout -b feature/new-vulnerability`)
3. **Add clear documentation** for any new vulnerabilities or attack vectors
4. **Include test cases** to demonstrate the vulnerability
5. **Update the README** with any new information
6. **Submit a pull request** with a clear description

## Reporting New Vulnerabilities

If you discover new MCP-related vulnerabilities:

1. Create a demonstration server similar to the existing ones
2. Document the attack vector clearly
3. Provide mitigation strategies
4. Include network analysis tools if applicable

## Code Standards

- Follow Python PEP 8 guidelines
- Include docstrings for all functions and classes
- Add comments explaining complex security concepts
- Ensure code is clean and well-documented

## Documentation

When adding new content:

- Update the main README.md
- Include technical details in appropriate sections
- Provide clear examples and use cases
- Explain both the attack and defense perspectives

## Questions?

Feel free to open an issue for any questions about contributing to this research project.
